# chronoq-bench

Benchmark harness for `chronoq-ranker`. Produces reproducible JCT vs load plots proving
LambdaRank beats FIFO on real traces. Zero runtime deps on the server or Celery.

**Status:** Chunk 2 in progress on `v2/chunk-2-bench`.

## What this package does

1. Loads job traces (BurstGPT from HuggingFace, or synthetic Pareto).
2. Replays them through a SimPy discrete-event simulator using pluggable schedulers.
3. Measures JCT metrics (mean/p50/p95/p99, HoL blocking, Jain's fairness) per scheduler.
4. Compares 6 schedulers: FCFS, SJF-oracle, SRPT-approx, random, priority+FCFS, LambdaRank.
5. Writes `bench/artifacts/results.json` + `jct_vs_load.png` (the README money plot).

## Layout

```
chronoq_bench/
├── __init__.py
├── metrics/
│   ├── jct.py          # mean/p50/p95/p99, HoL blocking count, Jain's fairness index
│   └── ranking.py      # Spearman ρ, Kendall τ, pairwise accuracy (wraps ranker helpers)
├── traces/
│   ├── base.py         # TraceJob dataclass + TraceLoader ABC
│   ├── synthetic.py    # Pareto-distributed generator (seeded, CI-safe, no download)
│   ├── burstgpt.py     # HuggingFace BurstGPT loader + parquet cache + schema validation
│   └── cache.py        # shared cache path helpers
├── simulator.py        # SimPy DES: Poisson arrivals, single-server, pluggable scheduler
├── baselines/
│   ├── fcfs.py
│   ├── sjf_oracle.py   # uses true_ms; non-preemptive
│   ├── srpt_oracle.py  # non-preemptive SRPT-approx (documented in BENCHMARKS.md)
│   ├── random_sched.py # name avoids shadowing stdlib random
│   └── priority_fcfs.py
├── experiments/
│   ├── jct_vs_load.py      # money plot: sweep load 0.3→0.9, 6 schedulers
│   ├── drift_recovery.py   # shift distribution, measure retrain recovery
│   └── ablation_features.py # leave-one-out feature sensitivity
└── plots/                  # matplotlib helpers (PNG + CSV output)
```

Artifacts land in `bench/artifacts/` (gitignored; regenerated by `make bench`).
Downloaded trace cache lives in `bench/data/` (gitignored; regenerated by `make bench`).
CI fixture sample lives in `bench/fixtures/burstgpt_ci_sample.parquet` (committed, 100 rows).

## Ownership

- **Simulator + baselines + experiments** → `benchmark-analyst`. Review via `/bench` output analysis.
- **Plots + BENCHMARKS.md** → `docs-writer`. Triggered after each experiment run.
- **Trace loaders** → `benchmark-analyst` for data integrity; `ml-engineer` for feature alignment.

## Key conventions

- All random seeds fixed: `numpy.random.seed(42)`, SimPy `random.seed(42)`. Results must be
  reproducible to ±2% across machines.
- `CHRONOQ_BENCH_OFFLINE=1` env var → use `bench/fixtures/burstgpt_ci_sample.parquet` (100 rows).
  CI always sets this. Full 188MB BurstGPT download only on developer machines.
- Pandas is allowed here (trace munging). It must NOT appear in `chronoq-ranker` runtime deps.
- `results.json` schema must include `feature_schema_version`, `n_features`, `seed`, `trace` (all four — any committed artifact missing these is invalid).
- SRPT is non-preemptive ("SRPT-approx"). Documented in `docs/v2/BENCHMARKS.md`.

## Running

```bash
make bench           # full harness (~10 min on 8-core laptop)
make bench-smoke     # 1k-record CI subset (<60s), uses CHRONOQ_BENCH_OFFLINE=1

uv run pytest tests/bench/ -v          # unit tests only (no simulation)
uv run ruff check bench/               # lint
```

## Exit criteria (Chunk 2)

| Metric | Target |
|---|---|
| LambdaRank mean JCT vs FCFS | ≥ 10% at load=0.7 |
| LambdaRank p99 JCT vs FCFS | ≥ 15% at load=0.5 AND load=0.7 |
| LambdaRank p99 vs SJF-oracle | within 20% at load=0.7 |
| `make bench` runtime | < 10 min on 8-core laptop |
| `results.json` | must contain `feature_schema_version` + `n_features` |
